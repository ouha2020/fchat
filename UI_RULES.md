# UI_RULES.md — HomeTree / FamilyChat UI 规则

本文档约束后续 UI 重构。它不授权修改业务逻辑、身份、权限、数据库、Push、Service Worker 或安全边界；涉及这些内容时必须回到 `AGENTS.md` 的项目治理规则。

## 产品方向

- 移动端优先，优先保障 360px、390px、430px 宽度体验。
- 体验参考 LINE、iMessage、Zenly，但保持 HomeTree / FamilyChat 自己的温暖家庭感。
- 视觉关键词：日本极简、原生 App 感、干净、柔和、可信、单手友好。
- 不做通用 SaaS 后台、不做企业管理系统、不堆叠大面积 dashboard 卡片。
- 避免过度渐变、过度阴影、过度装饰、AI 感很强的卡片堆叠。

## 不可破坏的体验边界

- 聊天页必须继续使用 `100dvh` / dynamic viewport 思路，不要改回固定 `h-screen`。
- 聊天底部输入栏、录音流程、工具栏、悄悄话提示条、重要通知条不能被 UI 改造破坏。
- iOS 键盘弹出、输入文字、语音录制时不能明显晃动。
- 日程页月 / 周 / 日切换要稳定，详情抽屉不能被键盘严重遮挡。
- Push 通知、Realtime、缓存、补偿同步、message recipient 可见性都不是 UI 装饰层，不能在 UI 重构中绕开。
- 长昵称、长标题、长备注、长家庭名必须截断或换行，不能撑破布局。

## 布局规则

- 默认 mobile first，再扩展到桌面窄容器；当前根布局使用最大宽度容器，不要轻易改成桌面后台布局。
- 页面主容器优先复用 `app-page` / `app-page-narrow` / `section-card` 等现有语义类。
- 触控目标建议不小于 40px，高频操作优先 44px 以上。
- 底部固定区域必须处理 `env(safe-area-inset-bottom)`。
- 抽屉、弹窗、选择器必须有最大高度和内部滚动，不能超出视口。
- 列表项、气泡、表单行要显式使用 `min-w-0`、`truncate`、`break-words`、`line-clamp` 或合理换行策略。

## 组件规则

- icon-only 按钮必须有 `aria-label` 或 `title`。
- 表单控件必须有关联 label，或通过 `aria-label` / `sr-only` 提供名称。
- 重要操作按钮文案要短，不要在按钮里塞解释性长句。
- 按钮、输入框、弹窗、toast、sheet、chips、badge 优先从现有 class 语义演进，不要每个页面重新发明一套样式。
- 没有引入图标库；当前图标主要来自 `public/ui-icons` 和少量文本符号。新增图标方案必须先在设计系统中登记。
- 卡片只用于真实分组、列表项、弹窗或工具面板；不要把每个页面区块都做成浮动卡片。

## 聊天页规则

- 消息正文、图片 URL、语音 URL、坐标、family code、member token、Auth token 不能通过 UI、URL、日志或 Push payload 暴露。
- 悄悄话必须在视觉上清晰标识，但不能让管理员默认读取别人之间的悄悄话。
- Realtime event 只是轻量信号，UI 不能假设 event 自带完整消息正文。
- 消息气泡要保持可读、可长按、可复制/撤回/高亮，不要牺牲现有手势。
- 语音、图片、位置消息的展示要保留失败态和安全 URL 校验。

## 日程页规则

- 私人日程只能在权限允许的人之间可见；UI 重构不能用前端过滤替代 RPC 权限。
- 日程详情、评论、负责人响应、提醒状态、活动记录必须保持一条闭环。
- 提醒 Push 只展示安全摘要；UI 不应引导把标题、备注、评论、位置、媒体 URL 放入 Push payload。
- 月 / 周 / 日视图切换、筛选、搜索和详情打开状态应稳定，避免布局跳动。

## 可访问性规则

- 所有可点击元素必须能键盘聚焦，焦点态不能被移除。
- 图标按钮、关闭按钮、前后切换按钮、展开/收起按钮必须有可读名称。
- 颜色不能作为唯一状态表达；状态应同时有文字、图标、形状或上下文。
- Toast、Dialog、Sheet 的交互应能被键盘完成；危险操作需要确认。
- 动效必须尊重 `prefers-reduced-motion`。

## 验收规则

- UI 代码改动默认执行 `npm run lint`、`npm run typecheck`、`npm run build`、`git diff --check`。
- 只改文档至少执行 `git diff --check`，并记录可用/缺失的脚本。
- UI 改动必须人工检查 360px、390px、430px，无横向滚动，无文字溢出，无关键按钮遮挡。
- 涉及 PWA、Push、键盘、录音、上传、位置、日程提醒时，必须补充对应真机或手动回归步骤。
