# Agent Reports

Worker agents must write report-only outputs in this directory.

Rules:

- Use one unique Markdown file per worker.
- Do not edit `PHASE_STATUS.md`, `TASKS_UI.md`, `docs/iteration-log/_latest.md`, pages, components, services, migrations, schema, Push, or Service Worker files.
- Reports should include scope, files reviewed, findings, risks, validation performed, and recommended next tasks.
- Reports are not authoritative until reviewed by the Orchestrator.
